# Actividades Semana 2

## Actividad 2.1 - Invectigación de Arquitecturas de Datos

Los data warehouse son sistemas que almacenan datos estructurados varios de varias fuentes que mantienen un registro histórico para permitir consultas rápidas y facilitar la analítica; mientras que los data lakes son repositorios que almacenan datos en crudo para exploraciones de datos, permitiendo escalabilidad e ingesta rápida de datos. Además. los data leaks, son las filtraciones de la información que en esas se guarda, así como también su publicación que a su vez expone información personal, datos, e información sensible la mayoría de las veces

| Data Warehouse | Data Lake |
| :---: | :---: |
| Almacena datos estructurados | Almacena datos en crudo |
| Sirve mejor para análisis de negocios | Es más utilizado por la ciencia de datos |
| Toma una mayor cantidad de recursos en el escalamiento | Son más económicos de almacenar y escalar |
| Siguen una estructura fuja y limitada | Convergen una mayor flexibilidad |
| Esquema "on-write" con una preparación de datos previa | Esqurma "on-read" con una preparación de datos a petición |

Un data mart es un subconjunto que se especializa en un data warehouse que se enfoca en una unidad principal de negocio que evita la necesidad de navegar por el almacén completo
